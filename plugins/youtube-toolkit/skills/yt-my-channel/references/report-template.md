# Report template — yt-my-channel

```markdown
# Channel report — @handle

## 1. Snapshot
- Subscribers: <N>
- Total views: <N>
- Total videos: <N>
- Created: <YYYY-MM-DD>
- Avg views per video (last 10): <N>
- Avg engagement ratio (likes/views): <X.X%>
- Posting cadence (last 12 weeks): <X videos/week>

## 2. Top performers
For each of the top 5 videos:
- Title — <views>, <likes>, <comments>, <duration>, published <date>
- Tags: <top 5 tags>
- Hook hypothesis: <one sentence — why it pulled views>
- Topic cluster: <flagship series / tutorial / lead-gen / experiment>

## 3. Underperformers
For the bottom 5 of the last 20 videos (exclude videos < 14 days old):
- Title — <views vs channel median>
- Likely cause: <topic mismatch / weak hook / niche overlap / format>

## 4. Patterns
- Length distribution: <histogram in words — e.g., "15–20 min dominates, but top 3 are 8–12 min">
- Title formulas reused: <list 2–4>
- Topic clusters by view share: <flagship: X%, tutorial: Y%, lead-gen: Z%, other: W%>
- Posting weekday pattern: <if any>

## 5. Hypotheses
- What works: <2–3 hypotheses with evidence>
- What does not: <2–3 hypotheses with evidence>
- Unknown / needs data: <list, e.g., "CTR by topic — need Studio paste">

## 6. Recommendations (next 4 weeks)
1. <concrete action — e.g., "Cut 15-min tutorials to 10 min; top 3 are <12 min">
2. <concrete action>
3. <concrete action>

## 7. Studio data (if provided)
- Date range: <…>
- Avg CTR: <X%>
- Avg view duration: <m:ss>
- Best CTR videos: <top 3>
- Worst retention videos: <top 3>
- Notes: <caveats from parsing>
```

## Writing rules

- Numbers must be exact, copied from MCP responses.
- Hypotheses labelled with the word "hypothesis" — never present them as fact.
- Russian only in quoted titles and in user-facing recommendation phrasing if the user prefers it; the analytical scaffolding is in English.
- Length cap: ~600 words total. If the channel has many videos, summarise — do not list every one.
