# Tag strategy (Russian IT / vibe-coding)

Tags are a relevance signal. They no longer drive direct discovery — but they help YouTube cluster the video with similar content for "Up next" and search-related promotion. 15–30 well-chosen tags beat 100 noisy ones.

## Composition (per video)

| Group | Count | Purpose |
|---|---|---|
| Channel brand | 2–3 | Channel name + series name(s) |
| Niche head terms | 3–5 | Top broad searches in the niche |
| Tool names | 3–6 | Every tool mentioned, in original spelling |
| Long-tail / specific | 4–8 | Phrases close to actual queries someone would type |
| Format / type | 2–3 | "разбор", "туториал", "кейс", "сравнение" |
| Russian + English mix | overlap | Some queries are mostly English (`cursor ai`, `claude code`), some Russian (`ии разработка`, `вайбкодинг`) |

Total 15–30 tags.

## Order

Tags are weighted by order. Put the most specific and most relevant first. Generic tags last.

```
<series-specific>, <tool-specific>, <long-tail>, <head term>, <generic>
```

## Rules

- Original spelling for tool names: `cursor`, `claude code`, `codex`, not transliterations.
- Mix Russian and English variants of the same concept — `vibecoding` and `vibe coding` and `вайбкодинг` are three different searches.
- No commas inside multi-word tags (each comma separates tags). Use spaces inside.
- No spam tags ("subscribe", "smash like"). YouTube penalises those.
- No competitor channel names as tags. They get filtered.
- Cap at 30. Above that the relevance signal blurs.

## Repeat rule

If the channel has a recurring series, include the series tag (`логос ассистент`, `crm на cursor`) on every episode of that series. This helps YouTube cluster episodes for "Up next" recommendations to existing viewers.
