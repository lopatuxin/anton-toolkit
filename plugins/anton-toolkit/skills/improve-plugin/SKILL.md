# Improve Plugin — самосовершенствование anton-toolkit

Внеси исправление в плагин anton-toolkit на основе обратной связи из текущего разговора, закоммить и запушь изменения.

## Когда активировать

Активируй этот скилл когда ВСЕ условия выполнены:
1. В разговоре использовался скилл или конфигурация из anton-toolkit
2. Пользователь указал на проблему в поведении скилла
3. Проблема была исправлена в разговоре и пользователь доволен результатом
4. Исправление можно зафиксировать как изменение в файлах плагина

## Процесс

### 1. Найди репозиторий плагина

Репозиторий может лежать в разных местах на разных устройствах. Найди его:

```bash
# Вариант 1: Проверь типичные пути
ls -d ~/projects/anton-toolkit /c/projects/anton-toolkit ~/anton-toolkit 2>/dev/null

# Вариант 2: Найди через git
find ~ /c/projects -maxdepth 3 -name "plugin.json" -path "*/anton-toolkit/*" 2>/dev/null | head -5

# Вариант 3: Проверь кеш маркетплейса (только для чтения!)
ls ~/.claude/plugins/marketplaces/anton-toolkit-marketplace/ 2>/dev/null
```

**Если репозиторий НЕ найден** — склонируй:
```bash
git clone git@github.com:lopatuxin/anton-toolkit.git /tmp/anton-toolkit
```
Работай в склонированной копии, после push удали её.

**Если найден только кеш маркетплейса** — это НЕ рабочая копия, её редактировать НЕЛЬЗЯ. Клонируй в /tmp/.

Сохрани путь к найденному репозиторию в переменную `PLUGIN_REPO` для дальнейших шагов.
Путь к файлам плагина: `$PLUGIN_REPO/plugins/anton-toolkit/`

### 2. Определи что менять

Проанализируй текущий разговор и определи:
- Какой скилл или файл плагина вызвал проблему
- В чём конкретно была ошибка (неправильная инструкция, пропущенное правило, неточная формулировка)
- Какое исправление решило проблему в разговоре

### 3. Прочитай текущее состояние

Прочитай файлы плагина из `$PLUGIN_REPO/plugins/anton-toolkit/`:
- Если проблема в скилле — прочитай соответствующий `skills/<skill-name>/SKILL.md`
- Если проблема в конфигурации — прочитай `plugin.json` или другой конфиг
- Прочитай все связанные файлы, чтобы понять контекст

### 4. Обнови версию плагина

ОБЯЗАТЕЛЬНО при каждом изменении инкрементируй версию в
`$PLUGIN_REPO/plugins/anton-toolkit/.claude-plugin/plugin.json`.

Правила версионирования (semver):
- **PATCH** (0.1.0 → 0.1.1): точечные исправления скиллов, правки формулировок, мелкие фиксы
- **MINOR** (0.1.1 → 0.2.0): добавление нового скилла или значительная переработка существующего
- **MAJOR** (0.2.0 → 1.0.0): кардинальная перестройка плагина (крайне редко)

Без обновления версии маркетплейс не подтянет изменения на другие устройства.

### 5. Внеси минимальное точечное исправление (помимо версии)

Правила редактирования:
- Вноси МИНИМАЛЬНОЕ изменение, которое решает конкретную проблему
- НЕ рефактори остальной код скилла заодно
- НЕ добавляй «улучшения» сверх того, что просил пользователь
- НЕ меняй стиль или форматирование незатронутых частей
- Формулируй новые правила конкретно — с примерами правильного и неправильного поведения

### 6. Покажи изменения пользователю

Перед коммитом покажи пользователю что именно изменилось (включая новую версию) и спроси подтверждение.

### 7. Закоммить и запушь

После подтверждения пользователя:

```bash
cd $PLUGIN_REPO
git pull origin main
git add <изменённые файлы>
git commit -m "<сообщение на русском>"
git push
```

Если клонировал в /tmp/ — удали после push:
```bash
rm -rf /tmp/anton-toolkit
```

Сообщение коммита должно кратко описывать что исправлено в скилле и почему. Например:
- `Убрал строку Co-Authored-By из шаблона коммита в скилле commit`
- `Добавил правило не использовать git add -A в скилле commit`
- `Исправил триггерные фразы в скилле improve-plugin`

## Важные правила

- ВСЕГДА показывай diff пользователю перед коммитом
- ВСЕГДА спрашивай подтверждение перед git push
- ВСЕГДА делай git pull перед коммитом — на другом устройстве могли быть изменения
- Если не уверен какой файл менять — спроси пользователя
- Не создавай новые файлы без явной необходимости
- Если изменение затрагивает несколько скиллов — делай один коммит на все связанные изменения
- НИКОГДА не редактируй файлы в ~/.claude/plugins/cache/ или ~/.claude/plugins/marketplaces/ — это кеш, изменения перезатрутся
