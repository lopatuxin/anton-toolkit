# anton-toolkit-marketplace

Личный маркетплейс плагинов для Claude Code. Каждый плагин решает свою задачу: от инженерных автоматизаций (коммиты, ревью, дебаг, devops) до проектирования систем, ведения YouTube-канала и личной стратегии.

## Структура маркетплейса

```
anton-toolkit-marketplace/
├── .claude-plugin/
│   └── marketplace.json        # манифест маркетплейса со списком плагинов
├── plugins/                    # все плагины маркетплейса
│   ├── anton-toolkit/          # инженерный набор: коммиты, ревью, дебаг, devops, dev-агенты
│   ├── personal-strategist/    # личная стратегия: онбординг, профиль, советник
│   ├── plugin-builder/         # создание и расширение плагинов в этом маркетплейсе
│   ├── system-designer/        # проектирование систем (concept → architecture → modules → roadmap)
│   └── youtube-toolkit/        # ведение YouTube-канала (контент, SEO, превью, аналитика)
└── docs/
    └── superpowers/            # внутренняя документация по подходам
```

### Анатомия одного плагина

```
plugins/<plugin-name>/
├── .claude-plugin/
│   └── plugin.json             # имя, версия, описание плагина
├── skills/                     # skills — вызываются по триггер-фразам или явно через /<имя>
│   └── <skill-name>/
│       ├── SKILL.md            # описание скилла и инструкции для модели
│       └── references/         # подробные референсы (шаблоны, чеклисты, правила)
├── agents/                     # subagent'ы — автономные исполнители (.md файлы)
│   ├── <agent-name>.md
│   └── references/             # общие референсы для агентов
└── commands/                   # slash-команды (если есть)
```

**Skills** запускаются в основном диалоге, ведут интервью с пользователем, генерируют артефакты. **Agents** — самостоятельные исполнители, которым можно делегировать задачу одним вызовом (например, сгенерировать код или провести ревью). У каждого агента — своя система промптов и часто свой набор инструментов.

## Установка

Маркетплейс уже подключён к Claude Code, плагины активируются через `/plugin`.

Репозиторий: `https://github.com/lopatuxin/anton-toolkit.git`

## Плагины

### 1. `anton-toolkit` (v0.18.2)

Инженерный швейцарский нож: коммиты, ревью, дебаг, devops, агенты-разработчики под Java/Kotlin/Python/Frontend, генерация тестов и QA.

#### Skills (`/skill-name` или по триггер-фразам)

| Скилл | Триггеры | Что делает |
|-------|----------|------------|
| **commit** | "commit", "коммит", "закоммить", "сохрани изменения" | Анализирует изменения, формирует коммит-сообщение на русском, делает `git commit`. |
| **claude-md** | "claude.md", "создай/обнови/почисти claude.md", "аудит claude.md" | Создаёт, правит, чистит и синхронизирует `CLAUDE.md` под проект с соблюдением лимитов и формата. |
| **debug** | "баг", "ошибка", "не работает", "падает", "тормозит", "почему не работает" | Систематический дебаг: гипотезы, локализация, root-cause analysis, фикс. |
| **devops** | "docker", "compose", "разверни", "задеплой", "nginx", "контейнер", "ci/cd", "пайплайн" | Контейнеризация, деплой, инфраструктура. Содержит шаблоны Dockerfile и compose в `references/`. |
| **feature-planner** | "спланируй фичу", "распланируй фичу", "продумай фичу", "уточни нюансы", "вот тз" | Интервью по требованиям ДО реализации. Итог — детальный план, по которому работают dev-агенты. |
| **improve-plugin** | "улучши плагин", "обнови скилл", "исправь агента", "запомни это в плагине" | Точечная правка скилла/агента в плагине маркетплейса с bump'ом версии и git push. |

#### Agents (вызываются Claude'ом автоматически или через `Agent`-tool)

| Агент | Когда вызывается |
|-------|------------------|
| **java-dev** | Любая правка файлов внутри Java-модуля (есть `pom.xml`) — `.java`, SQL, Spring XML, Velocity, HTML/CSS/JS в `src/main/`. |
| **kotlin-dev** | Любая правка файлов внутри Kotlin-модуля (есть `build.gradle.kts`) — `.kt`, `.kts`, SQL, Liquibase XML, YAML. |
| **python-dev** | Любая правка внутри Python-модуля (есть `pyproject.toml`/`setup.py`/`requirements.txt`) — `.py`, Alembic-миграции, Jinja, Dockerfile, конфиги. Работает по step-by-step плану. |
| **frontend-dev** | Любая правка `*.tsx`, `*.ts`, `*.jsx`, `*.js`, `*.css`, `*.scss`, `*.html`, ассетов в `public/`, фронтового `package.json`. |
| **code-reviewer** | "проверь код", "сделай ревью", "code review". Также автоматически после dev-агентов по глобальному правилу из `CLAUDE.md`. |
| **test-writer** | Проактивно после `java-dev`/`kotlin-dev`/`python-dev`/`frontend-dev` либо явно "напиши тесты для X". |
| **qa-engineer** | "протестируй фичу", "прогони smoke-тест", "протестируй перед мержем". Гоняет API через curl и UI через Chrome DevTools, выдаёт bug-report. |

### 2. `personal-strategist` (v0.3.0)

Личный стратегический советник: собирает информацию о пользователе, систематизирует, помогает строить стратегии достижения целей.

#### Skills

| Скилл | Назначение |
|-------|------------|
| **onboarding** | Первоначальный сбор данных о пользователе: цели, контекст, ограничения. |
| **profile-review** | Ревью и актуализация профиля пользователя. |
| **strategist** | Разработка стратегий на основе собранного профиля. |

#### Agents

| Агент | Назначение |
|-------|------------|
| **interviewer** | Ведёт интервью с пользователем, аккуратно вытягивает информацию. |
| **analyst** | Анализирует собранные данные, выявляет паттерны. |
| **observer** | Наблюдает за поведением и предпочтениями, фиксирует выводы. |
| **strategist** | Формирует стратегические рекомендации. |

### 3. `plugin-builder` (v0.2.0)

Создание и расширение плагинов в этом маркетплейсе через интервью с пользователем. Для точечных правок уже существующих скиллов/агентов — используется `improve-plugin` из `anton-toolkit`.

#### Skills

| Скилл | Назначение |
|-------|------------|
| **create-plugin** | Создание нового плагина с нуля: интервью → структура → манифест → первые скиллы/агенты. |
| **extend-plugin** | Добавление новых скиллов/агентов в существующий плагин. |

### 4. `system-designer` (v0.3.0)

Итеративное проектирование программных систем и модулей. Оркестрирует процесс: requirements → architecture → modules → roadmap → phase details. Производит документацию в `docs/` репозитория проекта. Кода не пишет.

#### Skills

| Скилл | Триггеры |
|-------|----------|
| **system-designer** | "спроектируй систему", "задизайним модуль", "опиши модуль X", "разбей на фазы", "сделай roadmap", "детализируй фазу". |

#### Agents

| Агент | Фаза процесса |
|-------|---------------|
| **architect** | Фаза 2 — пишет `docs/architecture.md` (компоненты, интерфейсы, data flow, стек). |
| **module-designer** | Фаза 3 — пишет `docs/modules/<name>.md` для каждого модуля. |
| **docs-updater** | Фаза 4 — пропагирует изменение дизайна по всем затронутым документам. |
| **roadmap-planner** | Фаза 5 — пишет `docs/roadmap.md`, разбивает систему на фазы. |
| **phase-detailer** | Фаза 6 — пишет `docs/phases/phase-NN-<slug>.md` с деталями для разработчика. |

### 5. `youtube-toolkit` (v0.3.0)

Набор инструментов для ведения русскоязычного IT/vibe-coding YouTube-канала. Канал выпускает 1 видео в неделю, до 20 минут, без Shorts.

#### Skills

| Скилл | Триггеры | Что делает |
|-------|----------|------------|
| **yt-my-channel** | "проанализируй мой канал", "топ видео на моём канале", "статистика канала" | Анализ собственного канала: что работает, что нет, топ-видео, метрики. |
| **yt-competitors** | "анализ конкурентов", "анализ ниши", "разбери канал X", "что заходит в нише" | Two modes: SINGLE (один конкурент, глубоко) и BATCH (скан ниши по каналам выше порога подписчиков). |
| **yt-ideas** | "идеи для роликов", "о чём снять", "темы для видео" | Генерация тем. **Hard requirement**: требует на вход отчёт от `yt-my-channel` или `yt-competitors`. |
| **yt-content-plan** | "контент-план", "расписание видео", "календарь канала", "контент на месяц" | Календарь публикаций с учётом cadence (1 видео/неделю). |
| **yt-script** | "сценарий для видео", "напиши сценарий", "hook для видео", "структура ролика" | Сценарий: hook, акты, CTA, темп. Максимум 20 минут. |
| **yt-seo** | "название и описание", "тайтл и теги", "подбери название", "теги для видео" | Заголовок (A/B варианты), описание с таймкодами, теги. Превью — НЕ здесь. |
| **yt-thumbnail** | "превью для видео", "обложка для видео", "thumbnail концепт" | Концепция превью: текст, композиция, цвет, эмоция. Картинку не рисует. |
| **yt-promo** | "продвижение канала", "как раскрутить", "лидген на канале", "конверсия в заказы", "закреп под видео" | On-channel lead generation: pinned-комменты, end-screens, CTA в видео/описании, воронка на сайт-визитку. Кросс-промо на других платформах — out of scope. |

## Где брать актуальные триггеры и детали

Все triggers и нюансы каждого скилла — в YAML-frontmatter его `SKILL.md`. У агентов — в frontmatter `.md` файла. Этот README — навигационная карта, а не источник истины.

## Где править плагины

- **Точечная правка скилла/агента** → скилл `improve-plugin` из `anton-toolkit` (правит исходник в этом репозитории, бампит версию, пушит).
- **Новый плагин** → скилл `create-plugin` из `plugin-builder`.
- **Новый скилл/агент в существующем плагине** → скилл `extend-plugin` из `plugin-builder`.
